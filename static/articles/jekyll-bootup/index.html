    Jekyll Bootup

    By Daniel Sand September 30, 2013 Permalink

    Over the time I tried many different various blog softwarestacks:

    • Wordpress
    • Drupal
    • Tumblr
    • own written ROR blog

    Each solution is independet and has a good working EcoSystem for coding updates plugins and usability. But Jekyll has one major plus point:

    It’s Static content

    No more databases, comment moderation, or pesky updates to install—just your content.

    Markdown (or Textile), Liquid, HTML & CSS go in. Static sites come out ready for deployment.

    Permalinks, categories, pages, posts, and custom layouts are all first-class citizens here.

    This sounds damn pretty - and it is ! :]

    Jekyll … and Hyde ?

    if you have ruby in place everything is set for go. use rvm - it makes you happy :)

    So lets get started:

    first we need the jekyll gem

    gem install jekyll

    generate a new empty static blog 

    jekyll new <sitenameddirectory>
